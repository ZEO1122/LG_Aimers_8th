# Environment Setup Guide (From Scratch)

이 문서는 아무런 개발 환경이 설정되지 않은 Ubuntu 시스템을 기준으로,
EXAONE 모델을 vLLM을 이용해 로컬에서 실행하기까지의 전체 환경 설정 과정을 설명한다.
본 프로젝트는 클라우드 API가 아닌 로컬 GPU 환경에서 대규모 언어 모델을 서빙하는 것을 목표로 한다.

⸻

## 1. 운영체제 및 기본 조건

본 프로젝트는 Ubuntu 20.04 이상 환경을 기준으로 작성되었다.
다른 Linux 배포판에서도 유사하게 동작할 수 있으나, 공식 지원 환경은 Ubuntu이다.

GPU를 사용한 추론을 권장하며, NVIDIA GPU가 장착된 시스템을 전제로 한다.
VRAM은 최소 16GB 이상을 권장한다.

⸻

## 2. NVIDIA 드라이버 및 CUDA 환경

GPU를 사용하기 위해서는 NVIDIA 그래픽 드라이버가 먼저 설치되어 있어야 한다.
드라이버 설치 후, nvidia-smi 명령이 정상적으로 동작해야 한다.

vLLM 및 PyTorch는 CUDA 런타임을 필요로 하지만,
CUDA Toolkit 전체를 직접 설치할 필요는 없다.
NVIDIA 드라이버에 포함된 CUDA 런타임만으로도 충분히 동작한다.

즉, “CUDA가 설치되어 있다”는 의미는
CUDA Toolkit을 수동으로 설치했는지가 아니라,
GPU 드라이버를 통해 CUDA 연산이 가능한 상태를 의미한다.

⸻

## 3. Python 설치

본 프로젝트는 Python 3.10을 기준으로 한다.
Ubuntu 기본 Python 버전이 3.10이 아닌 경우, 별도로 Python 3.10을 설치해야 한다.

Python 패키지 충돌을 방지하기 위해 가상환경 사용을 강력히 권장한다.

⸻

## 4. Python 가상환경 구성

Python 가상환경은 Conda 또는 venv 중 어느 것을 사용해도 무방하다.
이 문서에서는 Conda 사용을 기준으로 설명한다.

Python 3.10 기반의 새로운 가상환경을 생성하고 활성화한 후,
이후 모든 작업은 해당 가상환경에서 진행한다.

⸻

## 5. PyTorch 설치

vLLM은 내부적으로 PyTorch를 사용하므로 PyTorch 설치는 필수이다.

GPU를 사용하는 경우, CUDA를 지원하는 PyTorch 버전을 설치해야 한다.
CPU 전용 PyTorch도 설치는 가능하지만, 성능이 매우 느려 실사용에는 적합하지 않다.

PyTorch 설치 이후, Python 환경에서 torch 모듈이 정상적으로 import되는지 확인되어야 한다.

⸻

## 6. 필수 Python 패키지 설치

PyTorch 설치가 완료된 후, 본 프로젝트에서 사용하는 추가 패키지를 설치한다.

주요 패키지는 다음과 같다.
	•	vLLM: 고성능 LLM 추론 및 서빙 엔진
	•	transformers: Hugging Face 모델 인터페이스
	•	huggingface_hub: 모델 다운로드 및 인증 관리
	•	requests: API 호출을 위한 HTTP 클라이언트

이 저장소에 포함된 requirements.txt 파일을 통해 모든 의존성을 일괄 설치할 수 있다.

⸻

## 7. Hugging Face 계정 및 인증

EXAONE 모델은 Hugging Face Hub에 호스팅되어 있으므로,
모델 가중치를 다운로드하기 위해 Hugging Face 계정 로그인이 필요하다.

Hugging Face 웹사이트에서 Access Token을 발급받은 후,
로컬 환경에서 로그인 과정을 수행한다.

로그인이 완료되면 토큰은 로컬 캐시에 저장되며,
이후 모델 다운로드 시 자동으로 사용된다.

⸻

## 8. vLLM 서버 실행 준비

본 프로젝트는 vLLM을 서버 형태로 실행한다.
서버 실행을 위한 설정은 server 디렉토리에 포함된 실행 스크립트 파일에 정리되어 있다.

해당 스크립트는 다음 작업을 수행한다.
	•	EXAONE 모델을 Hugging Face Hub에서 다운로드 (최초 1회)
	•	모델을 GPU 메모리에 로드
	•	OpenAI 호환 API 서버를 지정된 포트에서 실행

모델 가중치는 저장소에 포함되지 않으며,
로컬 캐시 디렉토리에 자동으로 저장된다.

⸻

## 9. GPU 메모리 설정 주의사항

GPU 메모리가 제한적인 환경에서는 vLLM이 기본 설정으로 실행되지 않을 수 있다.

이 경우, 실행 스크립트에서
GPU 메모리 사용 비율을 낮추거나
최대 컨텍스트 길이를 줄이는 방식으로 조정할 수 있다.

이는 성능과 안정성 간의 트레이드오프이며,
대부분의 해커톤 문제에서는 축소된 설정으로도 충분하다.

⸻

## 10. 서버 실행 확인

vLLM 서버가 정상적으로 실행되면,
로컬 API 엔드포인트를 통해 로드된 모델 정보를 확인할 수 있다.

모델 목록에 EXAONE 모델이 표시되면 서버가 정상 상태임을 의미한다.

⸻

## 11. 클라이언트 테스트

서버 실행 후, client 디렉토리에 포함된 테스트 스크립트를 통해
간단한 텍스트 생성 요청을 보낼 수 있다.

이 테스트는 다음 사항을 검증한다.
	•	서버가 정상적으로 요청을 수신하는지
	•	모델이 정상적으로 텍스트를 생성하는지
	•	API 응답 형식이 올바른지

⸻

## 12. 백그라운드 서버 실행

서버를 장시간 유지하거나 터미널 종료 이후에도 실행 상태를 유지하기 위해
tmux 사용을 권장한다.

tmux 세션 내에서 서버를 실행하면,
세션을 분리하더라도 서버는 계속 동작한다.

⸻

## 13. 저장소 구성 원칙

본 저장소에는 다음 항목이 포함되지 않는다.
	•	모델 가중치 파일
	•	Hugging Face 캐시
	•	개인 인증 토큰
	•	가상환경 파일

이러한 항목은 모두 로컬 환경에만 존재해야 한다.

⸻
